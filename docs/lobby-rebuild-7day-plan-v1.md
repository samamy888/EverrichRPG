# Lobby Rebuild 7-Day Plan v1

目標：7 天內把大廳從「可跑」提升到「風格一致、空間可信、互動自然」。

## Day 1: 規則凍結
- 產出：
  - `docs/lobby-visual-constitution-v1.md`
  - `docs/lobby-spatial-grammar-v1.md`
- 動作：
  - 挑 3 個基準資產（店面/櫃台/柱）當視覺標尺
  - 鎖定主色盤與陰影規則
- 驗收：
  - 全員對規則無歧義
  - 後續調整都能引用規則條文

## Day 2: 空間骨架校正
- 主要檔案：
  - `src/data/facilities.ts`
  - `src/data/tpe2Layout.ts`
  - `src/scenes/TPE2LobbyScene.ts`
- 動作：
  - 先修主動線 + 店面帶 + 安檢帶的位置與碰撞
  - 清除會造成「看起來可走但走不過」的碰撞
- 驗收：
  - 兩條基準走線（縱向/橫向）不卡牆
  - 互動點不漂移

## Day 3: 資產一致性第一輪（最突兀 20%）
- 主要檔案：
  - `public/map/TPE2/props/*`
  - `src/data/tpe2Layout.ts`
- 動作：
  - 優先修最違和素材：色溫、亮度、縮放、錨點
  - 同類道具做批次統一
- 驗收：
  - 截圖看不出明顯「不同來源素材混搭」斷層

## Day 4: 資產一致性第二輪（整區統一）
- 動作：
  - 補齊前一天未處理區塊
  - 調整群組節奏（店面排布、候機區道具群）
- 驗收：
  - 上/中/下三段截圖都能維持同一風格

## Day 5: 互動體驗打磨
- 主要檔案：
  - `src/scenes/TPE2LobbyScene.ts`
  - `src/ui/UIOverlay.ts`
- 動作：
  - 微調互動半徑、提示時機、提示文案一致性
  - 確保互動提示與場景目標一致
- 驗收：
  - 靠近店面、安檢、登機門都能自然觸發提示與互動

## Day 6: HUD 與場景融合
- 主要檔案：
  - `src/ui/UIOverlay.ts`
  - `src/config.ts`
- 動作：
  - HUD 色彩/邊框/透明度與場景一致化
  - 保留可讀性但降低「覆蓋感」
- 驗收：
  - HUD 開啟後不破壞沉浸感
  - `npm run hud:test:quick` 持續通過

## Day 7: 回歸與封板
- 動作：
  - 固定機位 4 張對照圖（入口/安檢/店面/候機）
  - 逐項檢查 Day 1 規則是否違反
  - 清理臨時 debug 視覺
- 驗收：
  - 功能正常
  - 一致性顯著提升
  - 沒有新增明顯風格斷裂

## 每日固定檢查（必做）
1. `npm run hud:test:quick`
2. 目測固定機位截圖 4 張
3. 主動線來回步行一次（測碰撞與提示）
4. 記錄當日「最怪 3 點」與隔日修正項

## 風險與緩衝
- 風險：
  - 素材來源混雜，統一化會花比預期更久
  - 碰撞與視覺同步調整容易互相影響
- 緩衝策略：
  - 優先保主動線與店面帶體驗
  - 次要區域延後，不阻塞主體品質
