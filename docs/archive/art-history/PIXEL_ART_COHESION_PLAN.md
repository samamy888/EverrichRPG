# EverrichRPG 像素風一致性改造計畫

> 已完成的歷史計畫：目前正式入口為 [`../../ART_PIPELINE.md`](../../ART_PIPELINE.md)。

目標：以目前 `480×320` 遊戲畫面、`16×16` 地圖網格與 `40px` 角色顯示高度為基礎，建立可重複、可驗證的商用品質像素美術管線。

## 完成定義

- [x] Tiled 與 Phaser 顯示的素材、寬度、高度和錨點一致。
- [x] 遊戲用素材以目標顯示尺寸輸出，正常情況下不依賴任意比例的執行時縮放。
- [x] 角色、場景與物件遵守同一套透視、色盤、輪廓、光源和陰影規則。
- [x] 每張地圖具有清楚的主動線、視覺焦點、互動群組與留白。
- [x] 新資產通過自動檢查與標竿場景視覺驗收後才能進入正式地圖。

## P0：建立可信的地圖資料契約

- [x] 建立專用工作分支 `codex/pixel-art-cohesion`。
- [x] 建立本計畫與驗收清單。
- [x] Phaser 預設採用 TMJ `object.width` 與 `object.height`。
- [x] 保留 `displayHeight` 作為明確的特殊覆寫值。
- [x] 修正方向化腳本，不再硬編碼 tileset `firstgid`。
- [x] 驗證 TMJ object GID 解析出的 texture 與自訂 `texture` 屬性一致。
- [x] 修復現存 GID／texture 不一致物件。
- [x] 為尺寸契約與 GID 契約加入自動測試。
- [x] 通過 `npm run tiled:verify`、單元測試、型別檢查與正式建置。

### P0 驗收標準

- [x] Tiled 中替物件設定 `96×72`，Phaser 中即顯示為 `96×72`。
- [x] 執行驗證時，任何 GID／texture 不一致都會直接失敗並指出地圖、圖層與物件名稱。
- [x] 所有正式 TMJ 通過一致性驗證。

## P1：像素密度與尺寸級距

起始基準：86 個 Tiled 場景物件中，72 個不是來源尺寸 1:1 顯示，23 個使用 X／Y 不同倍率。完成後基準：89 個物件中 0 個使用 X／Y 不同倍率；保留的縮放皆為等比來源密度轉換。

- [x] 建立角色比例尺：40px 高角色、腳底基準線、頭頂安全區。
- [x] 定義物件寬度級距：16、32、48、64、80、96px；例外需記錄原因。
- [x] 定義小／中／大型物件的碰撞 footprint 級距。
- [x] 盤點所有執行時非 1:1 縮放素材，可用 `npm run pixel:scaling-audit` 重跑。
- [x] 將高解析來源離線處理為 2× 遊戲畫布，使用等比縮放、二值透明邊、無抖色母版量化與人工抽查。
- [x] 移除角色或職業專屬的臨時比例倍率。
- [x] 啟用 Phaser `roundPixels` 像素座標取整。
- [ ] 實機確認移動中沒有半像素閃爍或不可接受的低速頓挫。

### P1 驗收標準

- [x] 離線 100%、200%、300% 整數倍率輸出沒有不均勻像素。
- [x] 同類物件的尺寸差異已由 display size、source density 與 footprint 級距記錄。
- [ ] 鏡頭移動時輪廓不閃爍、不發生像素寬度跳動。

### 自動驗證

- `npm run pixel:verify`：資產准入、TMJ／TSJ、非等比縮放、單元測試與型別檢查。
- `npm run build`：正式建置。
- `npm run tiled:cohesion-refine`：可重跑全部第二輪構圖、結帳台對齊、地板變體與 firstgid 正規化。
- `npm run visual:qa`：以正式 TMJ／TSJ、display size 與 depth offset 重建 9 張離線預覽及總覽圖。
- 人工畫面驗收路線見 `docs/PIXEL_ART_VISUAL_QA.md`；需要使用者在實際顯示器確認的項目刻意保留未勾選。
