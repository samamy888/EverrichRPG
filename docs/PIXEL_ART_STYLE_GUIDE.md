# EverrichRPG 像素美術規格

本文件是正式資產的准入規格。視覺目標為 modern detailed pixel-inspired airport RPG；不是復古 16-bit，也不是縮小的平滑向量插畫。

## 專案尺度

| 項目 | 規格 |
| --- | --- |
| 遊戲畫面 | 480×320 |
| 世界網格 | 16×16 |
| 角色顯示框 | 40×40 |
| 主要資產來源密度 | 2× display density |
| 正式縮放倍率 | 優先 1、0.5、0.25；禁止 X／Y 不同倍率 |
| 錨點 | bottom-center |
| 預設放置 | baseline 與 footprint 對齊 16px 網格 |

## 視角模板

### South／正面

- 正面朝鏡頭，左右對稱。
- 垂直邊保持垂直，所有檯面與基座線保持水平。
- 只顯示淺薄頂面；禁止左或右側面單側大量露出。
- 資訊機台、正面櫃台與建築入口優先使用此模板。

### North／背面

- 與 South 共用寬度、總高度、底部錨點與頂面深度。
- 隱藏正面 UI，只保留背板、維修縫與必要結構。
- 不以縮放或水平翻轉假裝背面。

### East／West

- 兩方向必須由同一母版建立，互為結構鏡像。
- 側面可見，但仍維持 top-down RPG 視角，禁止 45° 強等角旋轉。
- 同一物件四方向的 footprint、檯面高度和像素密度一致。

## 輪廓與像素群集

- 外輪廓使用深藍或暖炭色，避免大面積純黑。
- 主要外輪廓以 1 個遊戲像素為基準；2×來源即 2 source pixels。
- 高光不可形成連續平滑向量邊；需要可讀、成組的像素階梯。
- 禁止單像素隨機雜訊與全表面平均抖色。
- 細節在 100% 遊戲尺寸下不可讀時應刪除，而不是增加對比。

## 光源與陰影

- 主光固定由左上方照射。
- 物體內部陰影偏暖灰或深藍，不使用純黑。
- 素材內不烘焙大面積地面投影。
- 落地 contact shadow 由遊戲依 collision footprint 統一生成。
- 壁掛物件使用 wallAttachment 陰影，不再疊加落地陰影。

## 材質語言

| 材質 | 規則 |
| --- | --- |
| 暖白塑料／石材 | 大面積乾淨，2–4 階暖白轉折，金色只作細邊 |
| 深藍科技設備 | 深藍主體、青色小面積發光，不讓整個外框發亮 |
| 金屬 | 亮邊集中在左上，暗階使用暖棕或灰藍 |
| 玻璃／螢幕 | 深藍底、少量青色與功能色塊，不放可讀小字 |
| 植物 | 中深綠成組，底座必須比葉片更接近場景科技材質 |
| 布料 | 降低高光，使用大色塊摺線，不使用塑膠般白色高光 |

## 資產輸出契約

- Prompt 必須和 raw 圖、processed 圖、最終圖一同保存。
- 最終圖包含 alpha、bottom-center 錨點與 manifest。
- manifest 至少記錄 `sourceSize`、`displaySize`、`tileFootprint`、`anchor`、`lightDirection` 和狀態。
- 正式資產在 Phaser 中只能等比縮放。
- TMJ 的 width／height 是實際顯示畫布，不是任意拉伸目標。
- TSJ texture、TMJ texture property 與 GID 必須一致。

## 准入檢查

- [ ] 100% 尺寸下輪廓清楚、細節沒有融成雜訊。
- [ ] 200% 與 300% 整數倍率下沒有非等寬像素。
- [ ] 視角符合 South／North／East／West 其中一個模板。
- [ ] 色彩可映射到 36 色母版。
- [ ] 光源來自左上，沒有獨立矛盾的地面陰影。
- [ ] 來源尺寸到顯示尺寸為等比倍率。
- [ ] footprint、錨點、碰撞與畫面接地位置一致。
- [ ] 放進標竿場景後，與角色、資訊機台、櫃台和植栽屬於同一世界。
