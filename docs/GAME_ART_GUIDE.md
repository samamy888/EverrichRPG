# 遊戲美術與資產製作規範

本文件是 V3 遊戲世界美術的唯一全域規範。個別地圖座標、碰撞區域與物件清單應保存為資料檔，不寫進本文件。

## 1. 目標與非目標

- 目標：讓地圖、建築、道具、角色與特效共享同一套空間、比例、材質及光影規則。
- 目標：產出的資產可以在遊戲中分層、互動、重用並單獨替換。
- 非目標：把一張完整插畫直接當成可遊玩的地圖，或直接裁切成帶有固定陰影的零散素材。

## 2. 美術方向

本節在第一張場景母版確認後定稿，至少記錄：

- 風格關鍵字與明確禁止事項
- 攝影機角度、透視方式與遊戲解析度
- Tile 尺寸及角色標準高度
- 色盤、材質、線條與細節密度
- 主光方向、環境光、陰影顏色與透明度
- 機場空間需要維持的品牌感與辨識度

在上述規格定稿前，不批次製作正式資產。

## 3. 場景母版

每一種主要環境先製作一張完整場景母版，作為視覺標準答案，但不直接作為 runtime 地圖。

母版必須能回答：

- 人物、門、櫃台、座椅與建築的相對比例
- 地板、牆面、玻璃、金屬與布料的材質語言
- 前景、中景、背景的遮擋關係
- 光線方向、接觸陰影與空間深度
- 玩家可行走區域和視覺焦點

## 4. 資產拆分規則

| 類型 | 建議形式 | 例子 |
| --- | --- | --- |
| 可重複平面 | Tile | 地板、地毯、低矮標線 |
| 可組裝結構 | Tile／Chunk | 牆面、轉角、柱子、玻璃隔間 |
| 大型場景物件 | 獨立 Sprite／Chunk | 櫃台、商店、登機門 |
| 小型重用物件 | 透明 Prop | 座椅、行李車、盆栽、垃圾桶 |
| 動態或互動物件 | 動畫 Sprite | 電子看板、自動門、手扶梯 |
| 角色 | 分動作 Sprite Sheet | 玩家、旅客、店員、工作人員 |
| 遮擋與效果 | 獨立圖層 | 柱頂、招牌、玻璃反射、燈光 |

需要碰撞、互動、Y-sort、動畫、替換或重用的物件不得烘焙進地面基底。大型或長條物件不得為了方便而塞進一般方形 Prop Pack。

## 5. 地圖圖層與 runtime 契約

建議基本順序如下：

1. 地面基底
2. 材質變化與地面標線
3. 牆體及固定結構
4. 大型場景物件
5. 可互動物件
6. 玩家與 NPC（依腳底錨點進行 Y-sort）
7. 前景遮擋
8. 統一燈光、反射與環境效果

碰撞、傳送點、事件區域、出生點與鏡頭邊界必須使用結構化資料，不從背景圖片像素推斷。

## 6. AI 資產生產流程

1. 產生並人工確認完整場景母版。
2. 從母版建立美術規格與 runtime 物件清單。
3. 使用 `generate2dmap` 產生純地面基底、Tile／Chunk、圖層資料、碰撞與 QA 預覽。
4. 使用 `generate2dsprite` 產生透明 Props、角色與動畫；每次生成都以已確認母版和地圖基底作為可見參考。
5. 動畫依 idle、walk、interact 等動作分開生成及驗收，再組成 runtime atlas。
6. 使用正式資產重建母版構圖，執行整合驗收。

所有接受的生成資產都必須保留 prompt 或 manifest 紀錄，以便重製與追蹤。

## 7. 命名與目錄契約

正式目錄在第一個 vertical slice 實作時依前端載入方式定稿。預計至少區分：

- `assets/maps/`：地面、地圖預覽與場景圖層
- `assets/tilesets/`：可重複 Tile 與結構模組
- `assets/props/`：獨立透明場景物件
- `assets/actors/`：玩家與 NPC 動畫
- `assets/fx/`：獨立特效
- `data/maps/`：擺放、碰撞、事件與場景 metadata

檔名使用小寫 kebab-case，禁止以 `final`、`new`、`v2` 等無法表達內容的名稱區分版本。

## 8. 驗收門檻

資產進入正式素材庫前必須通過：

- 與母版一致的視角、比例、色彩、材質與光源
- 透明邊緣乾淨，沒有背景殘色或裁切
- 腳底／底部錨點穩定，Y-sort 與接觸陰影自然
- Tile 可無縫重複，Chunk 接縫不可見
- runtime 物件、碰撞與視覺圖層互相對應
- 在實際遊戲鏡頭尺寸下仍清晰且不過度雜亂
- 使用正式 Tile、Chunk 與 Sprite 重建的整合畫面接近母版

任何單張素材看起來漂亮，但放入整合場景後不一致，仍視為未通過。

## 9. 第一個 Vertical Slice

第一階段只製作一個機場出境大廳遊戲畫面，用它確認：

- 最終美術方向
- 攝影機、Tile 與角色比例
- 地圖與 Sprite 生產流程
- 資產目錄及 manifest 格式
- 碰撞、互動、遮擋與 Y-sort 實作
- 母版重建驗收方式

通過後才擴充其他航廈區域，避免在規格未穩定時大量生產資產。
