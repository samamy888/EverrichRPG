# 像素一致性本地驗收路線

本地網址：`http://127.0.0.1:5173/`

離線總覽：`docs/visual-qa/all-regions-current.png`。若瀏覽器擷取暫時不可用，可執行
`npm run visual:qa`，以正式 TMJ、TSJ 與物件深度規則重建 9 張地圖預覽。
同一指令亦會輸出 `docs/visual-qa/scale-check/information-core-{100,200,300}.png`，
用於檢查整數倍率下的像素寬度；動態移動與輸入手感仍須在實機確認。

## 1. 安全檢查區

- 看左右兩側「安檢通道 A／B」櫃台。
- 兩者應為正視角鏡像，短端朝向中央通道。
- 櫃台輪廓不可斜轉，寬高不可拉伸，底部陰影應貼合 `80×32` footprint。

## 2. 出境大廳

- 看長型數位資訊亭與電動步道：動畫各幀尺寸不可跳動。
- 看左右植栽、廣告柱、燈柱與等候座椅：輪廓不可被壓扁或拉高。
- 玩家從物件前後通過時，遮擋順序應依腳底 baseline 自然切換。

## 3. 機場設施區

- 看自助點餐機、飲水機、販賣機、回收站、清潔推車與手扶梯。
- 所有物件應保持原比例；動態物件各幀底部錨點不可漂移。

## 4. 旅客服務中心（標竿場景）

- 上方新版服務櫃台是主要焦點。
- 左上植栽分隔服務區，左下新版航廈地圖位於藍色辨識區。
- 右下等候座椅形成用途明確的等候區；中央至右側出口保持可讀通道。
- terrazzo、ivory、carpet-blue 應有細微變化但不可搶過人物與互動物件。

## 5. 免稅商店區

- 入口上方導引牌、中央服務櫃台與左右排隊欄杆應形成單一入口焦點。
- 三間店的 POS、結帳商品與櫃台表面必須連成同一組，不可漂在牆面。
- 四座展示島的商品要壓在台面上，並保留中央十字主動線。

## 縮放檢查

- 依序檢查瀏覽器 100%、200%、300%。
- 像素邊緣應保持等寬，不可出現 X／Y 不同倍率造成的扁化。
- 移動時注意輪廓閃爍、半像素跳動、動畫尺寸漂移與接地陰影脫離。

## 自動驗證已涵蓋

- 每張地圖的每個出生點可抵達所有出口。
- 東／西現役方向資產的來源體積、平均色彩、光源與透明面積一致。
- 玩家深度在物件 baseline 前後會由後景切換至前景。
- 相機使用 `startFollow(player, true, 1, 1)`，並與全域 `roundPixels`、`pixelArt` 設定共同啟用整數像素渲染。
