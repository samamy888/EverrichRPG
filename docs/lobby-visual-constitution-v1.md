# Lobby Visual Constitution v1

目的：把「大廳像拼貼」的問題轉成可檢查、可執行的規則，讓所有地圖/道具/HUD 走同一套語言。

## 1) 世界觀與畫面基調
- 場景主題：機場室內、明亮但偏冷白光，乾淨、秩序、可閱讀。
- 觀感目標：寫實平面圖轉像素化，不是卡通誇張，也不是寫實貼圖混搭。
- 光源方向：上方略偏左；陰影方向固定往右下。

## 2) 色彩規範
- 主色群：
  - 地面基色：低飽和淺灰藍
  - 結構色（牆/柱）：中性灰藍
  - 導視色（招牌/關鍵交互）：暖金或機場藍
- 禁止：
  - 同區塊同時出現高飽和暖色 + 高飽和冷色互搶
  - 道具原圖色偏差超過場景主色盤太多（看起來像不同遊戲）
- 最低對比：
  - 可互動物件邊緣需能在 1 秒內被看見（不靠文字提示）

## 3) 材質與邊緣語言
- 描邊規則：
  - 結構件：細描邊或半描邊，不能粗黑邊
  - 軟裝道具：弱描邊，優先靠明暗分層
- 陰影規則：
  - 只用一種陰影邏輯（方向、強度一致）
  - 禁止同畫面同時出現「完全無陰影」和「高對比硬陰影」資產
- 顆粒規則：
  - 像素密度一致，避免某些素材過銳、某些過糊

## 4) 比例與縮放規範
- 基準資產（不可隨意改）：
  - `floorplan-shop`
  - `floorplan-checkin-counter`
  - `floorplan-column`
- 其他 `prop-*` 以基準資產視覺比例校正，縮放遵守同一區間。
- 同類物件（例如同排店面模組）縮放差異不可讓人肉眼覺得不是同系列。

## 5) HUD 融合規範
- HUD 邊框與場景導視共用同一色系（暖金 + 灰藍）。
- HUD 背板透明度固定在可讀範圍，不能壓過場景主體。
- HUD 元件字體/粗細與場景 UI 統一，避免「像外掛面板」。

## 6) 驗收門檻（每次改圖必過）
- 4 張固定機位截圖（入口/安檢/店面/候機）視覺一致性通過人工檢視。
- `npm run hud:test:quick` 通過。
- 新增或替換資產前後對比，不能讓一致性下降。

## 7) 不符合時的優先修正順序
1. 先修「比例」與「亮度」。
2. 再修「飽和度」與「色溫」。
3. 最後才修細節紋理。
