# Phase 6：探索導覽與旅客護照

## 目標

參考經典掌機 RPG 的探索節奏，在現有三店購物循環之上加入：

1. 機場導覽地圖
2. 旅客護照／收藏圖鑑
3. 隱藏紀念章

本階段不新增航班、戰鬥或時間流逝系統。

## 核心循環

```text
進入新區域
→ 導覽地圖登錄區域
→ 與店員／旅客互動
→ 護照登錄人物
→ 調查不起眼的場景物件
→ 發現隱藏紀念章
→ 提升區域探索完成率
```

## 功能規格

### 機場導覽

- 顯示五個現有區域的連接關係。
- 標示目前位置。
- 未曾到訪的區域以未知狀態呈現。
- 顯示各區域隱藏紀念章收集狀態。

### 旅客護照

- 記錄到訪區域。
- 記錄遇見的店員與旅客 NPC。
- 顯示已購買商品種類。
- 顯示隱藏紀念章與整體探索完成率。

### 隱藏紀念章

- 藏在既有場景物件中，不增加發光提示。
- 第一次調查時取得並寫入自動存檔。
- 再次調查時回到原本物件功能。
- 第一批共五枚，每個區域一枚。

## 資料與存檔

- 新增獨立探索存檔：`${saveKey}-exploration`。
- 儲存 `visitedRegionIds`、`metNpcIds`、`collectibleIds` 與目前區域。
- UI 僅讀取探索服務，不自行維護重複狀態。

## 驗收條件

- 進入區域後，導覽地圖立即更新。
- 與 NPC 互動後，旅客護照立即更新。
- 五枚隱藏紀念章皆可取得且不重複。
- MENU 可切換導覽與護照頁面。
- 重載頁面後探索進度保留。
- `npm run build`、Tiled 與商店資料驗證通過。
