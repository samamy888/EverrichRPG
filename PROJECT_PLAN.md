# Everrich RPG 重建計畫

## 現況

專案已回到 architecture-only baseline。此階段不包含任何舊版玩法或美術內容。

## 下一階段順序

1. 定義新版遊戲核心體驗與不可變設計原則。
2. 建立領域模型與 API contract；先寫 migration，再接 UI。
3. 建立最小垂直切片，驗證前端、API、MySQL 與後台資料流。
4. 確認美術方向後才建立資產管線與正式內容。
5. 每個功能需通過自動測試與 CI/CD 才能擴張。

## 暫不進行

- 搬回任何舊角色、地圖、商店、任務或商品資料
- 在核心體驗未確認前大量產生美術資產
- 為假設中的需求建立通用框架
